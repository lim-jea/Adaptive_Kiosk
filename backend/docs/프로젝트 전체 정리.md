# Adaptive Kiosk - 프로젝트 전체 정리

이 문서는 현재 백엔드 코드의 모든 내용을 정리한 것이다.

---

## 1. 프로젝트 개요

디지털 취약계층(고령층 등)이 키오스크를 더 쉽게 사용할 수 있도록 돕는 지능형 음료 주문 키오스크 시스템의 **FastAPI 백엔드**.

### 개발 단계

```
1단계 (현재): 기본 키오스크 기능
              메뉴 조회 → 옵션 확인 → 주문 확정

2단계: 지능형 기능
       비전 UI 전환 / 추천 / 음성 주문

3단계: 고도화
       CF 추천 모델 / 세션 로그 분석 / 관리자 대시보드
```

### 핵심 기능

| # | 기능 | 상태 |
|---|------|------|
| 1 | 기본 주문 (메뉴 조회 → 옵션 → 주문) | **1단계 완성** |
| 2 | 멀티 키오스크 관리 (X-API-Key) | **완성** |
| 3 | 비전 기반 UI 전환 | 2단계 (코드 있음, router 비활성) |
| 4 | 음료 추천 (플로팅 패널) | 2단계 (코드 있음, router 비활성) |
| 5 | 음성 주문 (패턴 매칭 + Gemini) | 2단계 (코드 있음, router 비활성) |
| 6 | 로그 기반 정량 평가 | **완성** (관리자 Basic Auth) |

## 2. 기술 스택

| 분류 | 기술 |
|------|------|
| Framework | FastAPI (전체 async) |
| ORM | SQLAlchemy 2.0 async (aiomysql) |
| DB | MySQL (TiDB Cloud — 도쿄 리전) |
| 인증 | HTTP Basic Auth (관리자) + X-API-Key (키오스크) |
| AI | Google Gemini Flash Lite (2단계 음성 주문) |
| 패키지 관리 | uv (pyproject.toml) |

### DB 선택 경과

| DB | 리전 | 쿼리 지연 | 연결 지연 | 비고 |
|----|------|-----------|-----------|------|
| 로컬 MySQL | - | ~1ms | ~5ms | 가장 빠름. 시연용 |
| Aiven (무료) | 방갈로르 | **180ms** | 1,374ms | 느려서 부적합 |
| **TiDB Cloud (무료)** | **도쿄** | **42ms** | 443ms | **현재 사용 중** |

- 팀원 간 DB 공유를 위해 클라우드 DB인 **TiDB Cloud (도쿄)** 를 사용 중
- 추후 속도가 느릴 경우, 로컬 MySQL로 전환하거나 유료 플랜으로 업그레이드 가능
- `.env`의 `DATABASE_CONN` 값만 변경하면 즉시 전환 가능

### DB 연결 설정

- `pool_size=5`, `max_overflow=5` (클라우드 무료 동시 연결 제한 대비)
- `pool_pre_ping=True` (끊긴 연결 자동 복구)
- `pool_recycle=600` (10분마다 연결 재활용)
- SSL 자동 감지: URL에 `aivencloud` 또는 `tidbcloud` 포함 시 SSL 활성화
- Windows: `WindowsSelectorEventLoopPolicy` 적용 (aiomysql + SSL 호환)

## 4. 인증 구조

| 대상 | 방식 | 처리 |
|------|------|------|
| Swagger 문서 | HTTP Basic Auth | main.py 미들웨어 |
| 관리자 API (analytics, kiosk register/list) | HTTP Basic Auth | Depends(verify_credentials) |
| 세션 시작 | X-API-Key 헤더 | Depends(get_current_kiosk) |
| 그 외 (메뉴, 주문 등) | 인증 없음 | session_uuid로 식별 |

## 5. 외부 식별자 (UUID)

내부 FK는 auto-increment integer, 외부 API는 UUID(hex 32자)로 식별.

| 테이블 | 외부 식별자 | 용도 |
|--------|------------|------|
| kiosk_sessions | `session_uuid` (32자 hex) | 세션 조회/종료/주문 시 참조 |
| orders | `order_uuid` (32자 hex) | 주문 조회 |
| kiosks | `api_key` (64자 hex) | 키오스크 인증 (UUID 역할 겸용) |

## 7. 기본 키오스크 API (1단계)

모든 Request는 Pydantic 스키마로 통제됨 (Field 제약 + Swagger 예시 포함).

### Kiosk
| 메서드 | 경로 | Request | 인증 | 설명 |
|--------|------|---------|------|------|
| POST | /kiosks/register | KioskRegisterRequest | Basic Auth | 키오스크 등록 |
| POST | /kiosks/verify | KioskVerifyRequest | 없음 | 기기 확인 |
| GET | /kiosks | - | Basic Auth | 목록 |

### Session
| 메서드 | 경로 | Request | 인증 | 설명 |
|--------|------|---------|------|------|
| POST | /sessions | - | X-API-Key | 세션 시작 → session_uuid 반환 |
| GET | /sessions/{session_uuid} | SessionGetRequest | 없음 | 세션 조회 |
| POST | /sessions/{session_uuid}/end | SessionEndRequest | 없음 | 세션 종료 |

### Menu
| 메서드 | 경로 | Request | 인증 | 설명 |
|--------|------|---------|------|------|
| GET | /menus/categories | - | 없음 | 카테고리 목록 |
| GET | /menus | MenuListRequest | 없음 | 메뉴 목록 (?category_name=) |
| GET | /menus/{menu_name} | MenuDetailRequest | 없음 | 메뉴 상세 + 옵션 |

### Order
| 메서드 | 경로 | Request | 인증 | 설명 |
|--------|------|---------|------|------|
| POST | /orders | OrderCreateRequest | 없음 | 주문 생성 (가격 재검증) |
| GET | /orders/{order_uuid} | - | 없음 | 주문 조회 |

### Analytics (관리자)
| 메서드 | 경로 | 인증 | 설명 |
|--------|------|------|------|
| GET | /analytics/sessions | Basic Auth | 세션 통계 |
| GET | /analytics/recommendations | Basic Auth | 추천 통계 |
| GET | /analytics/orders | Basic Auth | 주문 통계 |

## 8. 주문 흐름 + 가격 검증

```
프런트 (장바구니 = React state):
  unit_price = menu.price + Σ(선택 option_item.extra_price)
  total      = Σ(unit_price × quantity)

POST /orders 요청:
  {
    "session_uuid": "abc...",
    "items": [{
      "menu_name": "아이스 아메리카노",    ← name 기반
      "quantity": 2,
      "unit_price": 5000,                  ← 프런트 계산값
      "selected_options": [{ "option_item_id": 2 }]
    }]
  }

서버 처리:
  1. session_uuid → 내부 session_id 변환
  2. menu_name으로 메뉴 존재/판매중 검증
  3. option_item_id 존재 검증
  4. unit_price 서버 독립 재계산 (프런트 값과 비교)
  5. 옵션 스냅샷 저장 (이름, 추가금)
  6. total_price = Σ(서버 unit_price × quantity)
  7. 응답에 order_uuid, menu_name, option_name 포함
```

## 11. 실행 방법 + main.py 구조

### 서버 실행
```bash
cd backend
uv sync                                      # 최초 1회: 의존성 설치
uv run uvicorn main:app --reload --port 5000  # 서버 시작
```

- `uv`가 `.venv`를 자동 인식하여 실행 (별도 activate 불필요)
- `--reload`: 코드 변경 시 자동 재시작 (개발용)
- API 문서: http://localhost:5000/docs

### main.py 구조
1. **Windows 이벤트 루프**: `WindowsSelectorEventLoopPolicy` (aiomysql + SSL 호환)
2. **lifespan**: DB 커넥션 풀 초기화 → 테이블 생성 → 시드 데이터
3. **CORS**: localhost:3000, 3001, 5000, 8000, 8080
4. **docs 보호**: `/docs`, `/redoc` Basic Auth (openapi.json 제외)
5. **라우터**: 기본 API만 등록 (2단계는 주석 처리)
6. **헬스 체크**: `GET /`, `GET /health`

## 13. 향후 작업

### 2단계 — 지능형 기능
- 비전 모듈 서버 연동 (POST /vision/analyze)
- 추천 트리거 3단계 (default → tab_dwell → menu_select)
- 음성→메뉴 파싱 (matched_menu_ids)
- 세션 abandoned 자동 처리
- session_events 테이블 (UX 타임라인 로그)

### 3단계 — 고도화
- CF 추천 모델 (KNN + OpenSurvey 데이터)
- analytics 기간/키오스크별 필터
- 관리자 메뉴/옵션 CRUD API
- 관리자 JWT 인증 (admins 테이블)

## 14. 계층 구조

```
main.py
  ↓
api/v1/endpoints/ (Request 스키마로 입력 통제)
  ↓
services/ (비즈니스 로직)
  ↓
crud/ (DB 접근)
  ↓
models/ (테이블)

schemas/ (Request/Response — 모든 계층에서 사용)
core/ (설정, DB, 보안)
scripts/ (시드 데이터)
```

새 기능 추가: **model → schema → crud → service → endpoint → router 등록**
