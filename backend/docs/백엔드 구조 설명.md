## 데이터 흐름

### 1단계: 키오스크 등록 + 세션 생성

```
관리자        → POST /kiosks (Basic Auth) → API 키 발급
              ↓
키오스크 시작 → POST /sessions (X-API-Key) → session_uuid 발급
```

### 2단계: 메뉴 탐색 + 주문

```
GET /categories            → 카테고리 목록
GET /menus                 → 메뉴 목록 (페이지네이션, 카테고리 필터)
GET /menus/{menu_name}     → 메뉴 상세 + 옵션 그룹 (옵션 모달용)
       ↓
POST /orders               → 주문 생성 (서버 가격 재검증 + 옵션 스냅샷)
       ↓
PATCH /sessions/{uuid}     → 세션 종료 { status: ended, end_reason: completed }
```

### 3단계: 얼굴 분석 (간편모드)

```
POST /face/analyze { session_uuid, frames: [base64...] }
       ↓
얼굴 분석 (InsightFace 또는 mock)
       ↓
vision_event 저장 + 세션 PATCH (estimated_age_group, estimated_gender, is_simple_mode)
       ↓
응답: { age_group, gender, age_est, should_use_simple_mode }
```

### 4단계: 분석 (관리자)

```
GET /analytics/sessions?start_date=...&end_date=...&kiosk_id=...
GET /analytics/recommendations
GET /analytics/orders
```

## 계층 구조

```
endpoints → services → crud → models
              ↑
           schemas (입출력 검증)
```

- **endpoints**: HTTP 요청 수신, Pydantic 검증, 인증 의존성 주입
- **services**: 비즈니스 로직 (가격 검증, 통계 집계, 얼굴 분석)
- **crud**: 순수 DB 접근 (단일 책임)
- **models**: SQLAlchemy ORM 테이블 정의
- **schemas**: 모든 계층에서 사용하는 입출력 형식

## 멀티 키오스크 구조

```
┌─────────────┐  ┌─────────────┐  ┌─────────────┐
│  키오스크 A  │  │  키오스크 B  │  │  키오스크 C  │
│  (React)    │  │  (React)    │  │  (React)    │
│  1층 로비    │  │  2층 매장    │  │  3층 휴게실  │
└──────┬──────┘  └──────┬──────┘  └──────┬──────┘
       │ X-API-Key      │ X-API-Key      │ X-API-Key
       └────────────────┼────────────────┘
                        ▼
              ┌──────────────────┐
              │   FastAPI 서버   │
              │   (백엔드 1대)   │
              └────────┬─────────┘
                       ▼
              ┌──────────────────┐
              │     MySQL DB     │
              └──────────────────┘
```

- 서버 1대만 운영, 각 키오스크는 React 프런트만 실행
- 모든 세션 기록에 kiosk_id가 포함 → 키오스크별 분석 가능
- 키오스크 인증: 등록 시 발급된 API 키를 `X-API-Key` 헤더로 전송

## 환경변수 (.env)

| 변수 | 용도 |
|------|------|
| `KIOSK_USERNAME` | Swagger 문서 및 관리자 API 접근 계정 |
| `KIOSK_PASSWORD` | Swagger 문서 및 관리자 API 접근 비밀번호 |
| `DATABASE_CONN` | SQLAlchemy async DB URL (`mysql+aiomysql://...`) |
| `GENAI_API_KEY` | Google Gemini API 키 (음성 주문, 2단계) |

> ⚠️ 실제 값은 `.env.example`을 참고하여 본인 환경에 맞게 작성하세요.

## 외부 식별자 (UUID)

| 테이블 | 식별자 | 용도 |
|--------|--------|------|
| `kiosk_sessions` | `session_uuid` (32자 hex) | 외부 API 식별 |
| `orders` | `order_uuid` (32자 hex) | 외부 API 식별 |
| `kiosks` | `api_key` (64자 hex) | 키오스크 인증 |

내부 FK는 auto-increment integer를 그대로 사용.

## DB 모델 변경 시 주의

SQLAlchemy `Base.metadata.create_all`은 **이미 존재하는 테이블의 컬럼을 변경하지 않습니다**. 모델을 수정한 후에는 테이블을 강제 삭제하고 재생성해야 합니다.

```sql
SET FOREIGN_KEY_CHECKS=0;
DROP TABLE IF EXISTS categories, kiosk_sessions, kiosks, menu_option_groups, menus,
                     option_groups, option_items, order_item_options, order_items,
                     orders, recommendation_events, vision_events, voice_conversations;
SET FOREIGN_KEY_CHECKS=1;
```

서버 재시작하면 새 스키마로 생성되고 시드 데이터가 자동 삽입됩니다.
