# 백엔드 구조 설명

## 디렉토리 구조

```
Adaptive_Kiosk/
├── main.py                    # FastAPI 앱 진입점, CORS, docs 보호 미들웨어
├── .env                       # 환경변수 (DB 접속, 인증 키 등)
├── pyproject.toml             # 의존성 관리 (uv sync)
│
├── core/                      # 핵심 설정
│   ├── config.py              # 환경변수 로드 (pydantic BaseSettings)
│   ├── database.py            # DB async 엔진, 커넥션 풀, 세션 팩토리
│   └── security.py            # HTTP Basic Auth (JWT는 주석 처리, 추후 활성화용)
│
├── models/                    # DB 테이블 정의 (SQLAlchemy ORM)
│   ├── kiosk.py               # Kiosk - 키오스크 등록/인증 (API 키 기반)
│   ├── session.py             # KioskSession - 키오스크별 이용 세션
│   ├── vision_event.py        # VisionEvent - 비전 분석 결과
│   ├── recommendation_event.py # RecommendationEvent - 추천 이벤트
│   ├── order.py               # Order, OrderItem - 주문
│   ├── menu.py                # Category, Menu - 메뉴/카테고리
│   └── voice_conversation.py  # VoiceConversation - 음성 대화 기록
│
├── schemas/                   # 요청/응답 스키마 (Pydantic)
│   ├── kiosk.py               # 키오스크 등록/확인/응답
│   ├── session.py             # 세션 생성/수정/응답
│   ├── vision.py              # 비전 결과 입력/간편모드 판단 응답
│   ├── recommendation.py      # 추천 요청/응답/클릭 로그
│   ├── order.py               # 주문 생성/응답
│   ├── analytics.py           # 통계 응답 + 메뉴/카테고리 응답
│   └── voice.py               # 음성 입력/AI 응답/대화 이력
│
├── crud/                      # DB 접근 (순수 CRUD만)
│   ├── kiosk.py               # 키오스크 등록/조회/인증
│   ├── session.py             # 세션 생성/조회/수정/종료
│   ├── vision.py              # 비전 이벤트 저장
│   ├── recommendation.py      # 추천 이벤트 저장/클릭 기록
│   ├── order.py               # 주문 생성/조회
│   ├── menu.py                # 메뉴/카테고리 조회
│   └── voice.py               # 음성 대화 저장/이력 조회
│
├── services/                  # 비즈니스 로직
│   ├── vision_service.py      # 간편모드 전환 판단 (연령+신뢰도)
│   ├── recommendation_service.py # 추천 생성 (your_picks + discovery)
│   ├── analytics_service.py   # 통계 집계 (세션/추천/주문)
│   └── voice_service.py       # 음성 처리 (패턴 매칭 → Gemini 분기)
│
└── api/v1/
    ├── router.py              # v1 라우터 통합
    └── endpoints/             # API 엔드포인트
        ├── kiosk.py           # 키오스크 등록(Basic Auth)/확인/목록(Basic Auth)
        ├── session.py         # POST /sessions (X-API-Key 인증)
        ├── vision.py          # POST /vision/{session_id}
        ├── recommendation.py  # POST /recommendations/request, /click
        ├── order.py           # POST/GET /orders
        ├── menu.py            # GET /menus, /menus/categories
        ├── analytics.py       # GET /analytics/* (Basic Auth 인증)
        └── voice.py           # 음성 주문 시작/처리/이력
```

## 인증 구조

```
┌─────────────────────────────────────────────────────────┐
│                    인증 방식 분류                         │
├─────────────────────┬───────────────────────────────────┤
│ HTTP Basic Auth     │ Swagger 문서 (/docs, /redoc)      │
│ (KIOSK_USERNAME/    │ 관리자 API:                       │
│  KIOSK_PASSWORD)    │   POST /kiosks/register           │
│                     │   GET  /kiosks                    │
│                     │   GET  /analytics/*               │
├─────────────────────┼───────────────────────────────────┤
│ X-API-Key 헤더      │ 키오스크 API:                     │
│ (키오스크 등록 시   │   POST /sessions                  │
│  발급된 API 키)     │                                   │
├─────────────────────┼───────────────────────────────────┤
│ 인증 없음           │ POST /kiosks/verify               │
│                     │ GET  /kiosks/{id}                 │
│                     │ GET  /health                      │
│                     │ 기타 키오스크 운영 API             │
├─────────────────────┼───────────────────────────────────┤
│ JWT (주석 처리)     │ 현재 미사용. 추후 관리자 JWT      │
│                     │ 인증 도입 시 활성화 예정           │
└─────────────────────┴───────────────────────────────────┘
```

## 데이터 흐름

### 키오스크 등록 및 인증

```
관리자        → POST /kiosks/register (Basic Auth) → API 키 발급
키오스크 시작 → POST /kiosks/verify                → 기기 확인 + 정보 반환
이후 모든 호출 → X-API-Key 헤더 포함                → 키오스크 식별
```

### 일반 주문 흐름 (한 사용자 기준)

```
사용자 접근 → POST /sessions (X-API-Key) → 세션 생성 (kiosk_id 기록)
                              ↓
비전 분석   → POST /vision/{session_id}  → 간편모드 판단
                              ↓
                    ┌── 일반 모드 → 터치 주문
                    └── 간편 모드 → 음성 주문 or 간편 터치 주문
                              ↓
추천 요청   → POST /recommendations/request → your_picks + discovery
                              ↓
주문 완료   → POST /orders → 주문 저장
                              ↓
세션 종료   → POST /sessions/{id}/end
```

### 음성 주문 흐름 (취약계층 감지 시)

```
간편모드 진입 → POST /voice/{id}/start     TTS 안내: "주문하시겠습니까?"
      ↓
사용자 음성  → POST /voice/process         STT 텍스트 수신
      ↓
            ┌── 패턴 매칭 성공 → 즉시 응답 (빠름)
            └── 실패 → Gemini Flash Lite (시나리오 기반)
      ↓
TTS 응답    → requires_followup=true면 반복, false면 종료
      ↓
대화 이력   → GET /voice/{id}/history      전체 대화 조회
```

### 분석 (관리자 Basic Auth 필요)

```
GET /analytics/sessions         세션 수, 간편모드 전환율
GET /analytics/recommendations  추천 클릭률, 주문 전환율
GET /analytics/orders           평균 주문 시간, 추천 경유 비율
```

## 계층 구조 (호출 방향)

```
endpoints → services → crud → models
              ↑
           schemas (입출력 검증)
```

- **endpoints**: HTTP 요청 받아서 services/crud 호출
- **services**: 판단 로직 (간편모드 여부, 추천 생성, 음성 처리, 통계 계산)
- **crud**: 순수 DB 읽기/쓰기
- **models**: 테이블 구조
- **schemas**: 요청/응답 데이터 형태 정의

## 멀티 키오스크 구조

```
┌─────────────┐  ┌─────────────┐  ┌─────────────┐
│  키오스크 A  │  │  키오스크 B  │  │  키오스크 C  │
│  (React)    │  │  (React)    │  │  (React)    │
│  1층 로비   │  │  2층 매장   │  │  3층 휴게실  │
└──────┬──────┘  └──────┬──────┘  └──────┬──────┘
       │ X-API-Key      │ X-API-Key      │ X-API-Key
       └────────────────┼────────────────┘
                        ▼
              ┌──────────────────┐
              │   FastAPI 서버   │
              │   (백엔드 1대)   │
              └────────┬─────────┘
                       ▼
              ┌──────────────────┐
              │     MySQL DB     │
              └──────────────────┘
```

- 서버는 1대만 운영, 각 키오스크는 프런트(React)만 실행
- 모든 세션 기록에 kiosk_id가 포함 → 키오스크별 분석 가능
- 키오스크 인증: 등록 시 발급된 API 키를 `X-API-Key` 헤더로 전송

## 환경변수 (.env)

| 변수 | 용도 |
|------|------|
| KIOSK_USERNAME | Swagger 문서 및 관리자 API 접근 계정 |
| KIOSK_PASSWORD | Swagger 문서 및 관리자 API 접근 비밀번호 |
| DATABASE_CONN | SQLAlchemy async DB URL |
| GENAI_API_KEY | Google Gemini API 키 (음성 주문용) |
