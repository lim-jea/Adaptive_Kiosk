# Adaptive Kiosk — 프로젝트 전체 정리

이 문서는 현재 코드(백엔드 + 프런트엔드)의 모든 내용을 정리한 것이다.

---

## 1. 프로젝트 개요

디지털 취약계층(고령층 등)이 키오스크를 더 쉽게 사용할 수 있도록 돕는 **지능형 음료 주문 키오스크 시스템**.

- 카메라로 사용자를 인식하여 UI를 자동 간소화
- 메뉴/옵션 조회 → 장바구니 → 주문 → 결제 → 세션 종료의 전체 흐름
- 추후 음성 주문, AI 추천 시스템 추가 예정
- 모든 상호작용을 DB에 기록하여 정량 평가 가능

### 개발 단계

```
1단계 (현재 완성): 기본 키오스크 + 멀티 키오스크 + 얼굴 분석(mock)
2단계: 음료 추천 / 음성 주문 / 비전 모듈 실 연동
3단계: CF 추천 모델 / 관리자 대시보드
```

### 핵심 기능

| # | 기능 | 상태 |
|---|------|------|
| 1 | 메뉴/카테고리/옵션 조회 (동적) | **완성** |
| 2 | 옵션 선택 모달 (동적 렌더링) | **완성** |
| 3 | 주문 생성 (서버 가격 재검증) | **완성** |
| 4 | 멀티 키오스크 (X-API-Key) | **완성** |
| 5 | 얼굴 분석 → 간편모드 자동 전환 | mock 모드 동작, InsightFace 미연동 |
| 6 | 분석 (관리자 Basic Auth) | **완성** |
| 7 | 음료 추천 | 2단계 (코드 있음, router 비활성) |
| 8 | 음성 주문 (TTS/STT + Gemini) | 2단계 (코드 있음, router 비활성) |

## 2. 기술 스택

| 분류 | 백엔드 | 프런트엔드 |
|------|--------|-----------|
| 언어 | Python 3.11+ | JavaScript (React) |
| 프레임워크 | FastAPI (전체 async) | React 19 + Vite |
| ORM / HTTP | SQLAlchemy 2.0 async (aiomysql) | axios |
| 스타일 | - | TailwindCSS |
| 인증 | HTTP Basic Auth (관리자) + X-API-Key (키오스크) | X-API-Key 헤더 자동 첨부 |
| 패키지 관리 | uv (`pyproject.toml`) | npm |
| AI (2단계) | Google Gemini Flash Lite, InsightFace | - |

### DB

- 로컬 MySQL / Aiven Cloud / TiDB Cloud 등 어디든 사용 가능
- `.env`의 `DATABASE_CONN`만 변경하면 즉시 전환
- `aivencloud` / `tidbcloud` 호스트는 SSL 자동 활성화

## 4. 인증 구조

| 대상 | 인증 방식 | 처리 위치 |
|------|----------|----------|
| Swagger 문서 (`/docs`, `/redoc`) | HTTP Basic Auth | 백엔드 `main.py` 미들웨어 |
| 관리자 API (analytics, kiosks 생성/목록) | HTTP Basic Auth | `Depends(verify_credentials)` |
| 세션 시작 (`POST /sessions`) | X-API-Key 헤더 | `Depends(get_current_kiosk)` |
| 그 외 (메뉴, 주문, 얼굴 분석 등) | 인증 없음 | session_uuid로 식별 |

키오스크 프런트엔드는 `axios` 인터셉터에서 모든 요청에 `X-API-Key` 헤더를 자동 첨부.

## 5. DB 모델

### kiosks
| 컬럼 | 설명 |
|------|------|
| id (PK), name, location, api_key (UNIQUE 64자), is_active, registered_at, last_seen_at | 키오스크 등록 정보 |

### kiosk_sessions
| 컬럼 | 설명 |
|------|------|
| id (PK), session_uuid (UNIQUE 32자), kiosk_id (FK) | 외부 식별자 분리 |
| started_at, ended_at, end_reason, status | 세션 라이프사이클 |
| is_simple_mode, estimated_age_group, estimated_gender, help_triggered | 비전 결과 + 간편모드 |

### categories / menus
| 컬럼 | 설명 |
|------|------|
| categories: id, name, display_order | 카테고리 |
| menus: id, name, category, price | 기본 정보 |
| icon_emoji, calories, serving_temperature, is_caffeinated, is_seasonal, sweetness_level, bitterness_level | 추천 시스템용 구조화 속성 |
| description, image_url, is_available | |

### option_groups / option_items / menu_option_groups
- 옵션 그룹과 아이템이 별도 테이블, M:N으로 메뉴에 연결
- `name`이 UNIQUE이므로 이름으로 식별 가능

### orders / order_items / order_item_options
| 컬럼 | 설명 |
|------|------|
| orders: id, order_uuid (UNIQUE 32자), session_id, total_price, used_recommendation, status | |
| order_items: order_id, menu_id, quantity, **unit_price (스냅샷)** | 주문 시점 단가 보존 |
| order_item_options: order_item_id, option_item_id, **option_name, extra_price (스냅샷)** | 옵션 변경/삭제에도 과거 주문 보존 |

### vision_events / recommendation_events / voice_conversations
- 2단계 기능용 로그 테이블 (현재 vision_events는 face_service에서 사용 중)

## 6. 외부 식별자 (UUID)

내부 FK는 auto-increment integer, 외부 API는 UUID hex 32자로 식별.

| 테이블 | 식별자 | 용도 |
|--------|--------|------|
| kiosk_sessions | `session_uuid` | 세션 조회/종료/주문 |
| orders | `order_uuid` | 주문 조회 |
| kiosks | `api_key` (64자) | 키오스크 인증 |

## 8. 주문 흐름 + 가격 검증

```
프런트 (React):
  unitPrice = menu.price + Σ(선택 option_item.extra_price)
  totalPrice = Σ(unitPrice × quantity)
  → 장바구니 실시간 표시

POST /api/v1/orders:
  {
    "session_uuid": "abc...32hex",
    "items": [{
      "menu_name": "아이스 아메리카노",
      "quantity": 2,
      "unit_price": 5000,             ← 프런트 계산값
      "selected_options": [{ "option_item_id": 2 }]
    }]
  }

서버 처리:
  1. session_uuid → 내부 session_id 변환
  2. menu_name 검증 (존재 + is_available)
  3. option_item_id 검증
  4. 서버 사이드 unit_price 재계산
  5. 옵션 스냅샷 저장 (option_name, extra_price)
  6. total_price = Σ(서버 unit_price × quantity)
  7. 응답에 order_uuid, menu_name, option_name 포함
```

## 11. 시연 흐름

```
LandingPage  → POST /api/v1/sessions  (X-API-Key)
              ↓ session_uuid 발급
CameraPage   → 5장 캡처 (평문 base64)
              ↓
AnalyzingPage → POST /api/v1/face/analyze
              ↓ age_group, gender, should_use_simple_mode
ResultPage   → 결과 표시 + 간편모드 표시
              ↓
KioskPage    → GET /api/v1/categories
              → GET /api/v1/menus (페이지네이션)
              → 메뉴 클릭 → GET /api/v1/menus/{name} → 옵션 그룹 모달
              ↓ 옵션 선택 → 장바구니
PaymentPage  → POST /api/v1/orders
              ↓ 서버 가격 재검증 + 옵션 스냅샷 저장
CompletionPage → PATCH /api/v1/sessions/{uuid}
                { status: "ended", end_reason: "completed" }
```

## 13. 개발 시 주의사항

### DB 모델 변경 시
SQLAlchemy `create_all`은 기존 테이블을 변경하지 않으므로 모델 수정 후에는 모든 테이블을 강제 삭제하고 재생성해야 합니다.

```sql
SET FOREIGN_KEY_CHECKS=0;
DROP TABLE IF EXISTS categories, kiosk_sessions, kiosks, menu_option_groups, menus,
                     option_groups, option_items, order_item_options, order_items,
                     orders, recommendation_events, vision_events, voice_conversations;
SET FOREIGN_KEY_CHECKS=1;
```

### 한글 메뉴 이름
URL 경로에 한글이 포함되면 프런트의 `encodeURIComponent()`로 인코딩 필요.
백엔드는 FastAPI가 자동 디코딩.

### Windows + aiomysql + SSL
`main.py`에서 `WindowsSelectorEventLoopPolicy`를 적용해야 SSL 핸드셰이크가 정상 작동합니다.
